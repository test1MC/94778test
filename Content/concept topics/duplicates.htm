Duplicates

The Duplicate page displays possible duplicate submissions. BCS uses the submission application information you enter to conduct multiple searches through current submissions. If the submission application information matches other submissions, BCS displays those possible duplicate submissions in the page. You can then decide if the submission application you are entering is a duplicate of a previously entered submission.

How it Works

Previously, users were conducting multiple searches for a single client to determine if duplicate submissions existed prior to entering a new submission. This method of searching for duplicate submissions was time-consuming and prone to errors.

BCS now conducts the search for duplicate submissions for you. BCS uses the client information you enter to initiate a search using the insured name and first address state as its initial search criteria.

BCS then searches all company submissions using the following criteria:

  • Submissions related to the client ID returned from the initial search. If the search returns multiple client IDs, at least one of the client IDs must match.
  • Submissions not already identified as duplicate submissions.
  • Submissions that have not been deleted.
  • Submissions with an effective date within one month of the effective date entered on the new application.

The above is the default search criteria within BCS. BCS can add new criteria based on your company needs.

If any existing submissions match the above criteria, BCS displays them in the Duplicates page.

If you select one of the potential duplicates from the list, BCS takes the information contained in that duplicate and fills in the fields for your new submission.

BCS does not check for duplicate submissions when you edit an existing application. It only checks for duplicates when you enter a new submission application.

Exact Duplicates

BCS can return an exact duplicate result if certain search criteria are met. If the submission application you enter matches the agency code and client ID, and the date falls within one month of another submission application, BCS returns it as an exact duplicate. If BCS finds an exact duplicate, it gives you the option to cancel your current application or continue to enter the application information.

Clear Duplicates

You can remove the duplicate association from a submission. When you edit a submission that has an associated duplicate submission, you can delete the association.

Add Duplicate to Existing Submission

You can add duplicate information to an existing submission. When you edit the submission, you an add the duplicate submission number in the Duplicate of field.

What do you want to do?

Continue with Current Application - No Duplicate

Use this procedure to continue processing the new application as a non-duplicate.

  1. Enter the application information on the Application tab.
  2. Click Continue.
  3. Select the This application is not a duplicate of any of these checkbox.
  4. Click Continue.
 Continue with Current Application - Duplicate

Use this procedure to continue processing the new application as a duplicate.

  1. Enter the application information on the Application tab.
  2. Click Continue
  3. Select the submission in the Duplicates Submission list for which this is a duplicate.
  4. Click Continue.
 Cancel Current Application - Exact Duplicate

Use this procedure to cancel a current application as an exact duplicate.

  1. Enter the Application information on the Application tab.
  2. Click Continue.
  3. Click Cancel to quit the application process.
