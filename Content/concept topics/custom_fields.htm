Custom Fields

The Custom Fields page allows you to edit existing custom fields or add a new custom field to certain pages in BCS. You can also sort existing fields so that they display in a different order on the corresponding page in BCS.

You must have the proper administrative rights and permissions to access this page.

You can setup custom fields to capture information not currently captured in BCS. For example, if you wanted to capture prior carrier information but your current configuration of BCS does not have a field for this information, you can setup a custom field of Prior Carrier and assign it to a category such as Submissions. When you save the custom field, BCS then adds the field to the Submission pages.

When you add a new submission, the Prior Carrier field displays and you can enter a value for the field. When you save the submission, the prior carrier information you entered is associated with that submission.

Custom fields can display on multiple pages in BCS. For example, Policy Symbol is a custom field for Submissions, all pages in BCS that has submission information will display the Policy Symbol field. The Policy Symbol field displays on the Submission page, the Application Submission page, and any other page that contains Submission information.

Existing Customization

When you click on the arrow for the Existing Customization field, a drop-down list of all the current custom fields for your company displays. You can select an item from the list to edit.

Some custom fields have been setup as a List type of customization. This means that the field displays a list when users click on it. You can edit the list in the List Options section of the page. You can also add a new option to display in the list.

New Customization

When you click on the New Customization button, the fields you use to setup a new custom field display.  You can select the appropriate parameters for the new field from the items in the fields displayed here.

Sort Customizations

When you click on the Sort Customization button, current custom fields display by category. You can set the order in which the fields display in BCS.

What do you want to do?

Edit an Existing Custom Field

Use this procedure to edit a custom field.

  1. Select a custom field from the Select an existing custom field to edit list.
  2. Edit the existing information for the custom field as needed
  3. Click Save Customization.
 Create a New Custom Field

Use this procedure to create a new custom field.

  1. Click Create New Customization.
  2. Enter a meaningful description of the new field in the Description field.
  3. Select the appropriate item from the Customization Usage list.
  4. Select the appropriate item from the Customization Type list.
  5. Enter a date when you no longer want the field to display in the Expiration Date field.
  6. Select whether or not the field is required from the Required list.
  7. Select whether or not you want the field to be visible when a user creates a new submission from the Visible on Add list.
  8. Select whether or not you want the field read only from the Read Only list.
  9. Click Save Customization.
 Sort Custom Fields

Use this procedure to sort the custom fields in each category.

  1. Click Sort Existing Customizations.
  2. Click on a field and drag it to the position in the list where you want it to display in BCS.
  3. Click Save Changes.
